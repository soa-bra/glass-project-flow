# أولا- تحليل الوضع الحالي  
أ. مراجعة مستودع Glass Project Flow للتعرف على الوحدات المطورة (خدمات الفوترة، الفرص، المصروفات، HR، HR-lite، العقود، KB، Surveys، Approvals، Event Bus).  
ب. تحديد الوحدات التي تعمل جزئياً أو بلا تكامل (الفوترة دون مبيعات، CRM دون تسويق، KB دون ربط، Surveys دون فعاليات).  
جـ. تحديد الوحدات المفقودة (المحاسبة، المبيعات، التسويق، الفعاليات، التوظيف، المشاريع، الجداول الزمنية، التخطيط، الاتصالات، الدعم الفني، الرواتب).  
  
# ثانيا- المتطلبات العامة  
أ. استخدام معرفات موحدة وطبقة Event Bus وWebhooks، مع منع الكتابة المباشرة.  
ب. دعم الريال السعودي (SAR) والدولار الأمريكي (USD).  
جـ. اعتماد SoaBra كمرجع للكيانات (Party Master).  
  
# ثالثا- خريطة الطريق  
أ. إعداد البيئة: إنشاء مشروع React مع Vite وTailwind وShadcn UI، تنظيم المجلدات، إعداد Zustand/Redux.  
ب. استكمال الوحدات الحالية: توسيع الفوترة، تحسين CRM، تعزيز المصروفات والموافقات، توسيع HR، ربط KB وSurveys، تفعيل Event Bus.  
جـ. بناء الوحدات المفقودة: محاسبة، مبيعات، Social Marketing & Marketing Automation، Events & Appointments، Recruitment & Referrals، Project & Timesheet & Planning، Discuss & VoIP، Helpdesk، Payroll.  
د. تصميم طبقة التكامل: إعداد قاعدة Supabase، تعريف عقود البيانات، بناء Event Bus مع Webhooks، تفعيل HMAC.  
ه. تصميم واجهات المستخدم: تطبيق Glassmorphism، بناء صفحات للوحدات، إضافة السحب والفلاترة والبحث.  
و. الاختبارات: كتابة اختبارات وحدية وتكاملية، قياس الأداء، إعداد CI.  
  
# رابعا- خطة السبرينتات  
أ. السبرينت الأول: إعداد البيئة، إنشاء البيانات الوهمية، تطوير لوحة المشاريع، إعداد المعرفات وEvent Bus.  
ب. السبرينت الثاني: توسيع الفوترة، تحسين CRM، تحسين لوحة المشاريع، اختبارات وحدية.  
جـ. السبرينت الثالث: بناء وحدة المبيعات، تطوير Project & Timesheet، إنشاء Helpdesk، تفعيل WebSocket.  
د. السبرينت الرابع: بناء Social Marketing وMarketing Automation، تطوير Events & Appointments، إنشاء Recruitment & Referrals & Payroll، تحسين الواجهة.  
ه. السبرينت الخامس: تطوير Planning وDiscuss وVoIP، إعداد التقارير، اختبارات أداء، الوثائق، إطلاق النسخة التجريبية.
